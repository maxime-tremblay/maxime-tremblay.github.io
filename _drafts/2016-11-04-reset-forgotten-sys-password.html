--- layout: post title: Reset Forgotten SYS Password date: '2016-11-04T09:22:00.000-04:00' author: Maxime Tremblay tags: modified_time: '2016-11-04T09:22:25.271-04:00' blogger_id: tag:blogger.com,1999:blog-267713902572011003.post-388006861023933669 blogger_orig_url: https://max-tremblay.blogspot.com/2016/11/reset-forgotten-sys-password.html --- If you ever need to run any script from the Apex installation folder, you will need to connect to the database using SYS as SYSDBA. If you happen to have forgotten SYS's password you are out of luck. Or at least it would The idea is to launch SQLPlus without loggin as a user but to connect as sysdba. If you're running the database on Windows, you will need to locate the sqlplus executable. If you installed it using the defaults, it should be under this directory: c:\oraclexe\app\oracle\product\11.2.0\server\bin cd c:\oraclexe\app\oracle\product\11.2.0\server\bin sqlplus /nolog connect /as sysdba alter user SYS identified by "newpassword";
